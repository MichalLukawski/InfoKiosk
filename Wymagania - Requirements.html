Prace nad aplikacją rozpoczęto od serii spotkań z przedstawicielami instytucji celem określenia wymagań funkcjonalnych i niefunkcjonalnych oprogramowania.

Wymagani

 

Wymagania funkcjonalne

 

  1. Mapa biblioteki (wysoki priorytet): Aplikacja powinna, na dedykowanej podstronie, oferować interaktywną mapę biblioteki, która umożliwia użytkownikom zrozumienie układu budynku i lokalizacji różnych pomieszczeń. Powinna być możliwość przełączania między mapami różnych pięter.
    1. Wyświetlanie mapy (wysoki priorytet): Aplikacja powinna oferować interaktywną mapę biblioteki, która umożliwia użytkownikom zrozumienie układu budynku i lokalizacji różnych pomieszczeń. Mapa powinna wyświetlać w pierwszej kolejności piętro, na którym znajduje się infokiosk, z oznaczeniem “tu jesteś” na mapie.
    2. Przełączanie między mapami (wysoki priorytet): Powinna być możliwość przełączania między mapami różnych pięter. Biblioteka posiada 4 kondygnacje, dlatego funkcjonalność powinna oferować opcję przełączenia pomiędzy mapami poszczególnych kondygnacji.
    3. Lista pomieszczeń (wysoki priorytet): Po prawej stronie mapy powinna znajdować się lista wszystkich pomieszczeń biblioteki, zawierająca nazwę pomieszczenia, jego numer. Lista powinna być podzielona na sekcje - piętra, a każde pomieszczenie powinno znaleźć się w odpowiedniej, dla jego usytuowania, sekcji.
    4. Interakcja z listą i mapą (wysoki priorytet): Każde pomieszczenie kliknięte na mapie lub na liście powinno się podświetlić (zmienić kolor). Dodatkowo, jeśli kliknięte na liście pomieszczenie znajduje się na innym piętrze niż piętro, którego mapa jest obecnie wyświetlana, wtedy wyświetlana mapa piętra powinna się zmienić na odpowiednią.
    5. Opis pomieszczenia (wysoki priorytet): Kliknięcie na liście lub mapie powinno wywołać pojawienie się opisu pomieszczenia lub piętra w komponencie po lewej stronie mapy. W opisie pomieszczenia powinny znajdować się informacje 
       o funkcjach jakie spełnia oraz ewentualnych godzinach otwarcia.
  2. Wyświetlanie aktualności (średni priorytet): Aplikacja powinna umożliwiać wyświetlanie aktualności dotyczących funkcjonowania biblioteki na dedykowanej podstronie. Aktualności mogą zawierać zdjęcia, filmy, audio. Aktualności powinny być prezentowane w formie tablicy postów, gdzie każdy post reprezentuje jedną aktualność. Użytkownicy powinni mieć możliwość przeglądania postów w kolejności od najnowszego do najstarszego. Dodatkowo, powinna być możliwość aktualizacji tablicy postów przez personel biblioteki, aby zapewnić, że prezentowane informacje są zawsze aktualne.
  3. Najczęściej zadawane pytania (średni priorytet): Aplikacja powinna umożliwiać dostęp do najczęściej zadawanych pytań z odpowiedziami na dedykowanej podstronie. Pytania powinny być prezentowane w formie listy, gdzie każdy element listy reprezentuje jedno pytanie. Po kliknięciu na pytanie, powinna się wyświetlić odpowiedź na to pytanie. Dodatkowo, powinna być możliwość aktualizacji listy pytań przez personel biblioteki, aby zapewnić, że prezentowane informacje są zawsze aktualne i odpowiadają na najczęściej pojawiające się pytania użytkowników.
  4. Wirtualna wycieczka (niski priorytet): Aplikacja powinna umożliwiać dostęp do przygotowanej już “Wirtualnej Wycieczki”, czyli interaktywnej galerii zdjęć wnętrza biblioteki.

 

Wymagania niefunkcjonalne

 

  1. Interfejs użytkownika: Aplikacja powinna oferować prosty, ale atrakcyjny interfejs użytkownika.
  2. Kompatybilność z urządzeniem: Aplikacja powinna być kompatybilna z urządzeniem Infokiosk Dorado, które jest wyposażone w komputer PC, ekran dotykowy 43", głośniki stereo, procesor Intel i5-10400 @ 2.90GHz, 16 GB RAM, dysk NvMe 512GB i system Windows 11 Pro.
  3. Logo i nazwa biblioteki: Każda strona aplikacji powinna wyświetlać na górze logo biblioteki i jej nazwę. Spod tego warunku wyjęty zostaje moduł „Wirtualna Wycieczka”
  4. Informacje kontaktowe: Na dole każdej strony powinien być wyświetlany pasek z informacjami kontaktowymi do odpowiedniego personelu biblioteki. Spod tego warunku wyjęty zostaje moduł „Wirtualna Wycieczka”
  5. Przycisk powrotu do ekranu głównego: Każda strona powinna oferować przycisk powrotu do ekranu głównego.

 

Requirements

 

Functional Requirements

 

  1. Library Map (High Priority): The application should offer an interactive library map on a dedicated page, which allows users to understand the layout of the building and the location of various rooms. There should be the ability to switch between maps of different floors.
    1.  Displaying the Map (High Priority): The application should offer an interactive library map, which allows users to understand the layout of the building and the location of various rooms. The map should first display the floor where the info kiosk is located, with a “you are here” marker on the map.
    2. Switching Between Maps (High Priority): There should be the ability to switch between maps of different floors. The library has 4 floors, so the functionality should offer the option to switch between maps of individual floors.
    3. Room List (High Priority): On the right side of the map, there should be a list of all library rooms, containing the name of the room and its number. The list should be divided into sections - floors, and each room should be in the appropriate section for its location.
    4. Interaction with the List and Map (High Priority): Any room clicked on the map or on the list should be highlighted (change color). Additionally, if the room clicked on the list is on a different floor than the floor whose map is currently displayed, then the displayed floor map should change to the appropriate one.
    5. Room Description (High Priority): Clicking on the list or map should cause the description of the room or floor to appear in the component on the left side of the map. The room description should contain information about the functions it performs and any opening hours.
  2. Displaying News (Medium Priority): The application should allow displaying news related to the functioning of the library on a dedicated page. News can include photos, videos, audio. News should be presented in the form of a bulletin board, where each post represents one piece of news. Users should be able to browse posts in order from newest to oldest. Additionally, there should be the ability to update the bulletin board by library staff to ensure that the information presented is always up-to-date.
  3. Frequently Asked Questions (Medium Priority): The application should allow access to frequently asked questions with answers on a dedicated page. Questions should be presented in the form of a list, where each list item represents one question. After clicking on a question, the answer to that question should be displayed. Additionally, there should be the ability to update the list of questions by library staff to ensure that the information presented is always up-to-date and answers the most frequently asked questions from users.
  4. Virtual Tour (Low Priority): The application should allow access to the already prepared “Virtual Tour”, which is an interactive gallery of photos of the library interior.

 

Non-Functional Requirements

 

  1. User Interface: The application should offer a simple but attractive user interface.
  2. Device Compatibility: The application should be compatible with the Infokiosk Dorado device, which is equipped with a PC computer, 43" touch screen, stereo speakers, Intel i5-10400 @ 2.90GHz processor, 16 GB RAM, NvMe 512GB disk, and Windows 11 Pro system.
  3. Library Logo and Name: Each page of the application should display the library’s logo and name at the top. This condition excludes the “Virtual Tour” module.
  4. Contact Information: At the bottom of each page, a bar with contact information for the appropriate library staff should be displayed. This condition excludes the “Virtual Tour” module.
  5. Back to Home Screen Button: Each page should offer a button to return to the main screen.
